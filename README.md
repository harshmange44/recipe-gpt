# Recipe GPT — Built From Scratch

> A character-level GPT transformer trained to hallucinate cooking recipes.
> Built from scratch in PyTorch to understand every component of the transformer architecture.

This project is a hands-on implementation of the GPT architecture (decoder-only transformer)
trained on a dataset of cooking recipes. The goal: understand transformers deeply enough to
build one from first principles, not just call a library.

---

## What this generates

After training, the model produces recipes like:

```
TITLE: Spicy Banana Walnut Risotto

INGREDIENTS: 2 cups arborio rice, 1 banana (overripe), 3 tbsp olive oil,
1 cup chicken stock, 1 tsp cayenne, fresh parsley to garbish

INSTRUCTIONS: Heat olive oil in a large saucepan over medium heat. Add the
banana and stir until golden brown, about 3 minutes. Pour in the stock and
bring to a boil. Reduce heat and season with salt and garbish generously.
<END>
```

The model has learned: recipes have a `TITLE → INGREDIENTS → INSTRUCTIONS → <END>` structure.
The hallucinations (`garbish` instead of `garnish`) are a char-level artifact — and make for
great blog content.

## Architecture

```
Character tokens (B, T)
    ↓ Token Embedding (vocab_size=~90 → n_embd=384)
    + Position Embedding (block_size=256 → n_embd=384)
    ↓ × 6 Transformer Blocks:
        Pre-LayerNorm
        → Multi-Head Self-Attention (6 heads × 64 dim)
           Q @ K^T / sqrt(64) → causal mask → softmax → @ V
        + Residual connection
        Pre-LayerNorm
        → Feed-Forward (384 → 1536 → 384, GELU)
        + Residual connection
    ↓ Final LayerNorm
    ↓ LM Head (384 → vocab_size)
    → Logits → cross_entropy loss
```

**~10M parameters.** Trains in ~30–60 minutes on Apple Silicon MPS.

## Quick Start

### 1. Setup

```bash
cd recipe-gpt/
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 2. Download & prepare data

```bash
python data/prepare.py
# Downloads ~20K recipes from HuggingFace
# Outputs: data/processed/train.bin, val.bin, meta.pkl
```

### 3. Train

```bash
python train.py
# Trains for 5000 iterations on MPS (~30-60 min)
# Saves checkpoints to out/ckpt.pt
# Saves loss log to out/loss_log.json
```

For a quick test run (2 minutes):
```bash
python train.py --max_iters 500 --n_layer 2 --n_embd 128
```

### 4. Cloud Training (Google Colab)

If you don't want to heat up your local machine, you can train for free on Google Colab's T4 GPUs.

1. Upload the `colab_training.ipynb` file to Google Colab.
2. Run the cells. It will automatically:
   - Mount your Google Drive to save checkpoints safely.
   - Clone this repository.
   - Run the training loop using CUDA and `torch.compile` for a massive speed boost.

### 5. Generate recipes

```bash
python sample.py
python sample.py --prompt "TITLE: Spicy" --temperature 0.8 --top_k 40
```

### 5. Visualize training

```bash
python visualize.py
# Saves to plots/: loss_curves.png, attention_heatmaps.png, char_distribution.png
```


## Key concepts (for the blog)

| Concept | Where | Intuition |
|---------|-------|-----------|
| Character tokenization | `data/prepare.py` | 90 unique chars → integers. Each char is one token. |
| Token + position embeddings | `model.py GPT.forward` | "What is this char?" + "Where is it in the recipe?" |
| Causal self-attention | `model.py Head.forward` | Each char looks at past chars to predict the next one |
| `Q @ K^T / sqrt(d_k)` | `model.py Head.forward` | Scale prevents softmax saturation |
| Causal mask | `tril` buffer in `Head` | No peeking at future tokens during training |
| Residual connections | `model.py Block.forward` | Gradient highways through deep networks |
| Pre-LayerNorm | `model.py Block.forward` | Applied BEFORE sublayers — more stable than post-LN |
| Cross-entropy loss | `model.py GPT.forward` | `-log P(correct_next_char)` — lower = model is more confident |
| Perplexity | `visualize.py` | `exp(loss)` — "how many equally likely next chars?" |
| Cosine LR schedule | `train.py get_lr` | Warmup → smooth decay prevents training instability |
| Weight decay separation | `model.py configure_optimizers` | Only matrices decay; biases/norms don't |

## Hyperparameters

| Param | Value | Why |
|-------|-------|-----|
| `block_size` | 256 | ~2–3 recipe lines of context |
| `n_layer` | 6 | 6 transformer blocks |
| `n_head` | 6 | 6 attention heads per block |
| `n_embd` | 384 | 64 dim per head × 6 heads |
| `dropout` | 0.2 | Regularization (small dataset) |
| `batch_size` | 64 | 64 sequences × 256 chars = 16K tokens/step |
| `learning_rate` | 1e-3 | Higher than GPT-2 (smaller model can afford it) |
| `max_iters` | 5000 | ~45 passes through the dataset |
